# Decisions

| # | Decision | Alternatives Considered | Rationale |
|---|----------|------------------------|-----------|
| D1 | Remove `gasless?: boolean` from ExactTonPayload, detect from opcode | Keep boolean, add opcode check too | PR #1455 spec has no `gasless` field. Opcode detection is authoritative — the opcode is covered by the signature, cannot be forged. Removes client/facilitator coupling. |
| D2 | USDT-only gasless (native TON stays external) | Gasless for everything | Native TON payers already have TON for gas. TONAPI only accepts jetton commissions. No use case for gasless native TON. |
| D3 | TONAPI primary, self-relay fallback | TONAPI only / Self-relay only | TONAPI eliminates facilitator TON spending. Self-relay fallback preserves availability if TONAPI is down. Best of both worlds. |
| D4 | signedBoc always external message | Raw body Cell for gasless | PR #1455 spec says signedBoc is always "Base64-encoded external message." TONAPI expects external message. Unifies parsing. |
| D5 | Raw HTTP calls to TONAPI (no SDK) | Use @ton-api/client SDK | 3 simple endpoints. Avoids peer dep conflicts (@ton/core version). Keeps dependency tree minimal. SDK is auto-generated and heavy. |
| D6 | Commission from /estimate at boot, refresh hourly | Per-transaction /estimate / Hardcoded | TON gas prices are governance-controlled (change rarely). Boot-time discovery + hourly refresh balances accuracy and performance. Per-tx /estimate adds 200ms latency. |
| D7 | relayAddress = TONAPI relay_address | relayAddress = facilitator address | PR #1455: commission goes to whoever relays. When TONAPI relays, commission goes to TONAPI. Facilitator doesn't charge separately (x402 philosophy: facilitator is a public good). |
| D8 | Auto-detect authType in client based on asset + relayAddress | Explicit gasless parameter | USDT + relayAddress present → gasless. TON native → direct. Transparent to integrator. No new API surface. |
| D9 | Hex BOC for TONAPI, base64 for payload | Always hex / Always base64 | TONAPI spec says `format: cell` = hex. x402 payload spec says base64. Each uses its native format. Facilitator converts base64→hex when calling TONAPI. |
| D10 | Gasless settlement timeout 60s (vs 30s for direct) | Same 30s timeout | Gasless has extra hop (gas proxy → wallet). Two block confirmations needed. 30s may timeout under load. |
| D11 | Fix USDT mainnet address to `0:b113a994b5024a16719f69139328eb759596c38a25f59028b146fecd **c3621dfe**` | Keep current `...436021ff` | PR #1455 uses `...c3621dfe`. TonAPI, Tonkeeper, and tonviewer all confirm the correct address ends in `c3621dfe`. Our current `...436021ff` is wrong — TonAPI returns HTTP 400 for commission estimates against it. Critical bug fix. |
| D12 | Circuit breaker on self-relay fallback | No circuit breaker / Disable self-relay entirely | Unbounded self-relay drains facilitator wallet when TONAPI is down. Circuit breaker (3 consecutive failures within 60s → OPEN for 300s) bounds the damage. Daily budget cap (1 TON/day ≈ 10 txs at 0.1 TON each) adds a second safety layer. Disabling self-relay entirely would cause full outage when TONAPI is down. |
| D13 | Allowlist for TONAPI relay_address | Trust any address from TONAPI | If TONAPI API key is compromised or the endpoint is MITM'd, a manipulated relay_address would redirect all commissions to an attacker address. An allowlist (configurable via `FacilitatorOptions.allowedRelayAddresses`) prevents this. The known TONAPI relay address is the default allowlist entry; operators can extend it. |
| D14 | Absolute commission cap (10 USDT) | No cap / Per-asset cap | Safety rail against TONAPI returning an inflated commission estimate. 10 USDT is generous compared to current gas costs (~0.06 USDT per tx). If commission exceeds cap, log a warning and use the cap value rather than rejecting the transaction. |
| D15 | Native TON = intentional superset of PR #1455 | Drop native TON / Align strictly with PR #1455 | PR #1455 describes Jetton-only gasless relay. We additionally support native TON payments via external auth + sendFile. This is additive, not contradictory. The gasless path is only triggered for USDT + relayAddress. Native TON continues to use the existing external-auth path. Documented as a superset of the PR #1455 spec. |
| D16 | Merge tonApiConfig into existing `FacilitatorOptions` (4th constructor param) | Separate 5th constructor param / New interface | The facilitator constructor already accepts a 4th `options` param that includes `tonApiKey` (used for emulation). Adding `tonApiEndpoint` to the same object requires no breaking API change and no new parameter position. Operators who already set `tonApiKey` get endpoint configurability for free. |
| D17 | AbortController timeout on TONAPI calls (10s default) | No timeout / Use Promise.race | AbortController is the standard fetch cancellation mechanism and propagates cancellation to the underlying TCP connection. 10s is generous for a well-provisioned HTTP API but short enough to prevent a hanging settle() call from blocking the Hono request queue. Timeout is configurable via `FacilitatorOptions.tonApiTimeoutMs`. |
